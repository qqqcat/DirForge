# DirForge 生产级优化升级计划与落地（2026-03）

## 目标与计划

### 1) 性能专项
- 引入更大规模基准树（massive_tree）并纳入阈值回归。
- 扫描/聚合改为并发流水线：多 worker 目录扫描与聚合线程解耦。
- 快照事件仅传增量+Top列表，避免每次复制完整 `NodeStore`。

### 2) 执行安全专项
- 真实删除前置校验：风险、存在性、类型、权限。
- dry-run 对照：执行前进行模拟校验对照。
- 失败重试（I/O 类）与可恢复审计尾部记录。

### 3) 平台专项
- 强化跨平台异常映射：超时、忙碌、权限等统一分类。
- 提供平台能力矩阵（capabilities）用于策略降级。

### 4) 可观测专项
- 指标命名统一为 `df.*`，明确采集周期。
- 诊断包引入结构版本号，并在 UI 增加自动导出入口。

## 实施结果

- 性能：新增 massive 基准树，扫描回归测试覆盖小树+大树；扫描链路升级为多线程目录 worker 池 + 聚合线程，显著降低主循环阻塞。
- 快照：扫描过程中不再携带全量存储快照，改为 Top 列表增量。
- 安全：删除执行新增重试、dry-run 对照、审计记录。
- 平台：新增能力矩阵 API，异常映射更细。
- 可观测：新增指标描述与采集周期，诊断输出包含遥测快照与审计尾部。

## 增量优化（本轮）

- UI 与扫描流：
  - UI 待处理批次/快照队列改为有界 `VecDeque`，并在超限时执行受控丢弃。
  - 增加 UI 背压指标（丢弃批次数/快照数）与帧计数，支持抖动定位。
- 扫描背压：
  - 扫描事件通道改为 `sync_channel` 有界队列，削峰并抑制事件堆积内存放大。
- 平台一致性：
  - 新增 `assess_path_access(path)`，统一输出规范化路径、目录属性、重解析点标识与权限边界（Allowed/ReadOnly）。
- 可观测与归档：
  - 新增系统级快照（PID、CPU 并行度、RSS）并接入诊断 JSON。
  - 新增 `export_diagnostics_archive`，支持按时间戳生成诊断归档目录。

## 降级策略（摘要）

- explorer/select 能力不可用时：仅显示路径，不发起系统调用。
- recycle bin 不可用时：界面仅允许 dry-run，不启用真实删除。
- stable identity 不可用时：去重回退至大小/哈希组合，禁用 inode 级特化路径。
