# DirOtter UI 组件规格（2026-04-09）

## 1. 当前产品结构

- 主路径页面：`Overview / Live Scan / Result View / Duplicate Files / Settings`
- 二级工具页面：`Errors / Diagnostics`
- 危险动作入口：右侧 `Inspector`
- 当前产品心智：克制、直接、高信息密度，但一切围绕“帮用户释放空间”

## 2. 页面壳体

固定 shell：

- 顶部工具栏
- 左侧导航
- 中央主内容区
- 右侧 Inspector
- 底部状态栏

布局原则：

- 主内容区必须使用明确的最大宽度与对称 gutter
- Overview 可以使用单独更窄的宽度约束
- 页面优先整体滚动，不依赖固定高度主卡强行裁切内容
- Inspector 必须拥有独立纵向滚动

## 3. 页面职责

### Overview

- 首页必须先回答“现在最值得做什么”
- 必须优先展示：
  - `一键提速（推荐）`
  - 清理建议摘要
  - KPI
  - 最大文件夹 / 最大文件证据区
- 扫描入口必须优先展示盘符快捷按钮，手动目录输入是次级入口
- 不能把底层批次、快照、并发参数直接暴露给普通用户

### Live Scan

- 显示实时进度、实时 Top-N、最近扫描文件
- 必须明确这是“实时增量视图”，不是最终结果
- 停止扫描必须通过 `Stop Scan` 进入安全收尾

### Result View

- 只在扫描完成后提供结果浏览
- 只展示当前目录的直接子项
- 支持逐层进入和返回上级
- 结果区必须吃满剩余高度
- 进入页面时不得在 UI 主线程同步解压或重建结果树

### Duplicate Files

- 目标必须是“帮助用户放心删除重复文件”，不是单纯列出重复项
- 检测必须先按 `size` 收敛候选，再在页面进入后后台补算 hash
- UI 必须按组展示，默认按可释放空间降序
- 每组必须自动给出 `推荐保留`
- 默认只删除非推荐项，且高风险组默认不自动选
- 页面必须支持折叠组、懒加载和打开所在位置

### Settings

- 展示语言、主题和 `高级工具` 开关
- 如果当前回退到临时会话存储，必须显式提示本次设置不会跨退出保留

### Errors / Diagnostics

- 默认不打断主路径
- 只在高级工具开启后显示
- Diagnostics 当前只展示当前会话结构化信息和维护动作

## 4. Inspector 规范

Inspector 始终服务当前选中对象。

必须包含：

- 名称
- 路径
- 大小
- 快速操作
- 当前后台任务或最近执行反馈
- 系统内存状态摘要

当前关键动作：

- `Open File Location`
- `Move to Recycle Bin`
- `Delete Permanently`
- `Fast Cleanup`
- `Release System Memory`

动作约束：

- `Fast Cleanup` 只能在低风险缓存目标上出现，不能长期以灰按钮占位
- 永久删除必须先过确认层
- 删除确认后窗口必须立即关闭，反馈转为后台任务态
- 删除失败的完整细节必须收口到详情视图，不得挤在外层 banner

## 5. 删除与反馈规范

- 后台删除线程必须逐项回传已处理、成功、失败统计
- 删除完成后的结果同步必须在后台进行
- `结果同步中` 必须是显式可见状态
- 回收站删除成功后应提示用户可从系统回收站恢复
- Windows 回收站删除要做可见性二次确认

## 6. 清理建议规范

- 必须是规则驱动，不依赖 AI 推断
- 风险映射必须直接进入 UI：
  - `Low` => 可清理
  - `Medium` => 谨慎
  - `High` => 手动处理
- 绿色项可默认勾选
- 黄色项默认不勾选
- 红色项必须锁定，不允许直接执行
- 详情窗必须支持完整列表审阅，而不是只显示前几项

## 7. 结果与恢复规范

- 结果默认只保留在当前会话内
- Result View 仅允许恢复当前会话快照
- 删除/结果同步期间，若结果树未驻留内存，页面应显示等待态
- 不允许在渲染函数内同步读取快照、解压快照或反序列化 `NodeStore`

## 8. 本地化与字体

- 当前支持 19 种语言选择
- `中文 / English / Français / Español` 为完整 UI 文案
- 其余语言当前允许英文回退
- 多脚本系统字体 fallback 必须优先保证 CJK、Indic、Thai 等场景不出现大面积 tofu

## 9. 当前仍需补强

1. 最小视觉回归
2. 更统一的页面节奏与细部栅格
3. 删除过程更强的阶段反馈
4. 多语言下的长期回归保护
